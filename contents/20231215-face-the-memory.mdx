## なんでそう思ったん

ふら〜とネットサーフィンしている時に[この記事](https://note.com/simplearchitect/n/nf436809ac327)に出会い、雷に打たれたような衝撃がビリビリ全身に走った。というのも、自分の中でモヤモヤしていた部分をこの記事の内容が明確に言語化してたから。

「なんで同僚や先輩のxxさんはこんなにキレキレでお仕事ができるんや〜？」とモヤモヤしている自分と同じ境遇の方は是非ともご一読あれ。

この記事の内容をなぞるように自分の感想を書くだけではあるが、自分自信を鼓舞するための記事として残しておく。

## 記憶を避けていた自分

> 記憶ができないので、たとえばどこかに書いておくとか自分の記憶を頼らず、極力「記憶」を避ける方向で回避していた

初っ端から、とても頭が痛くなる文章。

自分も仕事においてはギリギリ問題ないが(そう主観的に思ってるだけかもしれない)、私生活では問題だらけ、、、

（親なのに）色んなサービスの引き落とし日は忘れるし、「●●取ってきて。or 連絡しておいて。」と嫁によく頼まれるが、それも忘れる。（記憶を避けているというより、ただ「怠惰」な人間である）

だから以下のように何かしらにメモとして残すことで何とか様々な**記憶**を回避していたんだと思う。

- Notion や iOS標準リマインダーで スケジュール・ドキュメントの管理をする
- X で良質なツイートは忘れないようにブックマークに追加する
- Youtubeは見直したい動画はすぐプレイリストに追加する
- ブックマーク管理には `raindrop` を使用し、特に重要な部分には ハイライト もバチバチ活用する。

> 急ぎ癖があるので、他のことを同時にやって、生産性が落ちるミックス癖につながり、「理解しない」ことにつながり、「記憶」は時間がかかりそうだし、自信がないので、「紙に書いておく」事をする。だからいつまでたっても「コントロール」できている実感が生まれないし、アウトカムも生まれない。だからまた「急いでやらないと」と思い、、、

特にこの部分が自分の事を100%見透かされているようで胸がとても、、痛い。

自分はエンジニアとしては遅咲き(25歳から)のスタートだからこそ、1日でも早く社内、社外どちらにおいても通用するような技術者になりたいと言う焦りがあり、結局同時並行で色々手をつけてしまう。

しかし「**理解して記憶すること**」に自信は持てないため、いつでも思い出せるようにと様々な方法・媒体を用いてメモとして残していた。

まさに、その行為が今向き合っている一つの問題の本質を理解し記憶する事を妨げていたんだと思う。

> だから私のような人はそういった「記憶」や「緻密さ」が必要なくても良い職につくとずっと楽で楽しく行けるだろう。実際私はそれらの職で「努力」した覚えすらない

この発想は全くなかった。

現状の自分を変えることなく、最大限のバリューを出せる職種に行くのもいい手かもしれない。(自分のキャリアに悩んだり、エンジニアという職業に疑問を抱き始めたら、考えるのはアリかもしれない)

## 自分なりに「記憶」と向き合う

> Clues, Note, Summary の形式で自分のやった作業を記録して、その日の終わりに整理して、何をやったかを「記憶」しようと務めた。記憶するために、「自分がそれを人にそらで説明できるか？」ということを「記憶できた」のチェックポイントにすること

とてもプラスになりそうだと思ったので、以下の形式で実行してみることにする。（自分のスタイルに合わなければ辞める）

```tsx
(Topics)
// ここに今日学んだ内容をタイトルのみ記載する。

-> 次の日に、Topicsだけに目を通し、言語化できるか確認

(以下の2つは何か仕組みを作らないと難しそう)
-> 学んだ内容が過去のものと被った場合の対処法はどうする？
(過去のtopicsに対して全文検索・曖昧検索をかけて重複してないかをチェックする仕組み)
-> 長期記憶の定着を図るために、次の日だけでなく 1週間後・1ヶ月後 にもリマインドしたい。

(Note)
// Topicsの内容に対して、理解(人に対して言語化して説明できる)した上で、自分なりの言葉で説明を記載する

```

## 頭の中で整理、そしてその訓練をする

> 周りのイケてる同僚にもう一つ共通することは、「頭の中で考えを整理する」のが得意ということだ。

これは確かになと思った。

仕事を死ぬほどサバく人（1日が48時間あるタイプの人）は、一日MTGが立て続けにあって事前にアジェンダを確認できなくても、即座に議題の本質を理解し意見を出せるし、業務もそつなくこなしている。

レスポンスに関しても同様で正確だし超早い。
それは当たり前で、既に頭の中で整理 & 理解している状態のため相手に伝える事はまとまっているため、下書きに沿って文字を打っている感覚なのだろう。

> 最初の「ミーティングの議事をその場で書かない」だが、これも確か、先ほど述べた「Clues/Note/Summary」の整理法にのっていたことをヒントにしている。簡単なメモは取ってもいいが、そこに頼るというより、「**その場で記憶までする**」つもりで聴くと、「理解」しないと無理なので、明確でないことは確認する。頭で整理しながら聴く

ここで言う「理解」することとは、他人に説明できるレベルまで言語化できている状態にすることである。

MTG中にこの状態まで持っていくとなるとそこそこの集中力、そして意識を持つ「習慣」がないと厳しいだろう。

自分は癖でやってしまっていたが、MTG中に議事録を取っていたり、相手が話している時に、自分が話す内容などを考えている状態だとMTGの内容をその場で理解し、記憶まで持っていくのは不可能だな〜と感じた。

## 最後に

> **時間をかけて「理解」しないと、コントロールできるようにならず、頭の中で整理された状態で「記憶」しないと、何回も思い出す必要や調べなおす必要があるので、時間もかかり、これも自分に確信が持てなくなる**。最後に反復は、理解して、整理して頭に入ったら、あとはそれをいつでもできるように「訓練」することに時間をかけること。

すごく当たり前の事なのに、自分は全く出来ていない。

時間をかけて「理解」するフェーズを飛ばしてしまう余り、新しい知識が必要になった際に、それに付随する知識を思い出すために ドキュメントを漁り、issueを漁りと余計な時間を使うことになるのだ。

大した事はなさそうと思うのは大間違いで、この差は雪だるま式に大きくなることになる。

「記憶」にしっかりと向き合うことで、雪だるま式に イケてる & 仕事ができる人になれるのだと思う。
